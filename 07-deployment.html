Diagramme de déploiement

Catégorie : Structure

Présentation

Deployments capture relationships between logical and/or physical elements of systems and information technology assets assigned to them.

- Object Management Group

Le diagramme de déploiement est une vue statique qui sert à représenter l'utilisation de l'infrastructure physique par le système et la manière dont les composants du système sont répartis ainsi que leurs relations entre eux.

- Wikipedia

Principaux éléments graphiques

Nom Représentation Description
Frame Elément permettant d'identifier un diagramme par son label.
Node Permet d'identifier les éléments physiques du système (serveur web, serveur de base de données...).
Component Représente une application, un logiciel.

Principaux éléments graphiques

Nom Représentation Description
Artifact Représente une partie d'une application (un fichier texte, binaire, une table en base de données...).
Association Permet de relier des éléments entre eux.

Exemple

Exercice

  1. Créer un diagramme de déploiement pour la mise en ligne d'un site WordPress
  2. Il n'y aura qu'un seul serveur hébergeant les fichiers du site internet et la base de données MySQL
  3. Lors de la mise en ligne, il faut effectuer plusieurs modification (utilisez des notes pour les schématiser) :
    • Modifier les infos de connexion à la BDD dans le fichier wp-config.php
    • Modifier les lignes "site_url" et "home" dans la table "wp_options"
    • Mettre à jour le fichier .htaccess à la racine du site

Exercice

  1. Créer un diagramme de déploiement pour la mise en ligne d'une appli web / mobile
  2. L'application web sera déployée sur un serveur et utilisera la technologie ReactJS et le serveur web Apache
  3. L'application mobile sera déployée sur un autre serveur et utilisera la technologie ReactNative
  4. Ces deux technologies utilisant le gestionnaire de packet NPM, NodeJS devra être installé
  5. La commande npm run build permettra de générer l'application en production
  6. Le déploiement étant réalisé via Git, il devra également être installé sur ces serveurs
  7. Enfin, un 3ème serveur permettra de gérer la base de données MongoDB
