Diagramme d'activité

Catégorie : comportement

Présentation

An Activity is a kind of Behavior that is specified as a graph of nodes interconnected by edges.

- Object Management Group

Le diagramme d'activité est un diagramme permettant de représenter le déclenchement d'événements en fonction des états du système et de modéliser des comportements parallélisables. Le diagramme d'activité est également utilisé pour décrire un flux de travail (workflow).

- Wikipedia

Principaux éléments graphiques

Nom Représentation Description
Activity Elément permettant de regrouper un ensemble d'actions.
Action Elément permettant de représenter une tâche.
Initial Node

Représente le début d'un ensemble d'actions.
Activity Final Node

Représente la fin d'un ensemble d'actions.
Flow Final Node

Représente la fin d'un sous-ensemble d'actions.

Principaux éléments graphiques

Nom Représentation Description
Object Flow Représente les échanges entre les objets et actions.
Fork / Join Node Permet de joindre ou diviser un flux d'activités qui s'exécuteront en parallèle.
Decision / Merge Node

Représente une condition pouvant entraîner deux chemins différents.
Swimlane

Permet d'identifier les acteurs dans un diagramme d'activité.

Exemple

Diagramme d'état-machine

Le diagramme d'état-machine est similaire au diagramme d'activité mais il représente des états et non des actions.

Il représente les états intermédiaire définis par les actions du diagramme d'activité.

Exercice

Réalisez un diagramme d'activité pour un distributeur de boisson.

  1. L'utilisateur insert une pièce de monnaie dans le distributeur puis choisit un produit.
  2. Le produit est disponible ou non (si non, une demande de réapprovisionnement est envoyée).
  3. Si le produit est disponible, le distributeur vérifie si il y a assez de pièces (si non, l'utilisateur devra insérer plus de pièces).
  4. Le distributeur active ensuite le mécanisme et l'utilisateur peut alors récupérer le produit.

Exercice

Réalisez un diagramme d'activité pour le déroulement d'un projet informatique.

  1. Le client rédige une demande.
  2. Le chef de projet rédige le cahier des charges par rapport à cette demande.
  3. Le client valide le cahier des charges (si non, le chef de projet le modifie).
  4. L'équipe de développement créée l'application demandée.
  5. En parallèle, un manuel d'utilisation de l'application est rédigé.
  6. Des tests sont réalisés (si ils ne sont pas validés, retour à l'équipe de développement).
  7. L'application et le manuel d'utilisation sont envoyés au client.
