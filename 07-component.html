Diagramme de composant

Catégorie : Structure

Présentation

A Component represents a modular part of a system that encapsulates its contents and whose manifestation is replaceable within its environment.

- Object Management Group

Le diagramme de composants décrit l'organisation du système du point de vue des éléments logiciels comme les modules, les données ou encore les éléments de configuration. Ce diagramme permet de mettre en évidence les dépendances entre les composants (qui utilise quoi).

- Wikipedia

Principaux éléments graphiques

Nom Représentation Description
Frame Elément permettant d'identifier un diagramme par son label.
Component Permet d'identifier les éléments physiques ou logiciels du système.

Principaux éléments graphiques

Nom Représentation Description
Interface
  • Provided Interface : Interface schématisée par un cercle représentant une interface exposée.
  • Required Interface : Interface schématisée par un arc de cercle représentant une interface requise.
Port Permet d'indiquer qu'un composant n'inclue pas directement une interface mais utilise une classe interne pour exposer une interface.
Dependency Représente une dépendance entre deux composants.

Exercice

Représentez le diagramme de composant d'un système de gestion d'entreprise en respectant les règles suivantes :

  1. L'application est découpée en 2 grandes parties : Production et Comptabilité
  2. La partie Production contient un composant permettant de gérer les projets
  3. La partie Comptabilité contient un composant permettant de gérer les clients et un autre pour la facturation (devis + factures)
  4. Un projet doit être associé à un client et la génération de devis / facture est réalisée à partir des données d'un projet
